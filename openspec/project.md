# monacori — North Star

## 한 줄
**monacori는 사람이 한 코드 리뷰를, AI 에이전트에게 줄 프롬프트로 바꾸는 키보드 우선 리뷰 surface다.**
diff 뷰어가 아니다. 산출물은 "PR 승인"이 아니라 **"에이전트가 고칠 프롬프트"** 다. GitHub 리뷰가 human→human이라면 monacori는 **human→agent**다.

## 루프
```
        ┌──────────────────────── monacori ────────────────────────┐
  AI ──▶ │  diff/소스 리뷰  →  ?/> 코멘트(코드 캡처)  →  합본  →  복사 │ ──▶ AI(에이전트)
  코드   └───────────────────────────────────────────────────────────┘   "이거 고쳐"
```
- `?` = 질문, `>` = 수정요청. 드래그하면 **선택한 코드까지** 코멘트에 캡처되어 프롬프트가 자기완결적이 된다.
- `Cmd+Shift+?` / `Cmd+Shift+>` → 종류별 합본 뷰(`buildMergedText`) → 전체 선택·복사 → 에이전트 프롬프트.
- 모든 키보드 내비게이션(캐럿·단어이동·comment-box nav·hunk·viewed)은 **리뷰를 빠르게** 하기 위해서만 존재한다.

## 만든다 / 안 만든다 (테제 필터)
| 만든다 ✓ | 안 만든다 ✗ |
|---|---|
| 리뷰를 빠르게 — 키보드 내비, 변경 접기, 변경에 집중 | 풀 IDE 편집·디버깅·리팩터 |
| 프롬프트를 좋게 — 코드 캡처, 구조화, 루프 닫기 | 멀티유저 human↔human 협업 리뷰 (GitHub 몫) |
| 가볍고 빠르게 — **제1원칙** | 리뷰 속도·프롬프트와 무관한 장식 |

## 이 문서 쓰는 법
새 change/기능은 두 질문을 통과해야 한다 — **(1) 리뷰를 더 빠르게 하나? (2) 프롬프트를 더 좋게 하나?** 둘 다 아니면 의심하라. `config.yaml`의 `context`에 압축본이 들어 있어 `/opsx:propose`·`/opsx:apply` 시 AI에게 자동 노출된다.

## 현재 베팅 (2026-06-20 기준 — 가변)
1. **루프 닫기** — 합본 뷰를 복사-붙여넣기 없이 에이전트로 전달(프롬프트 파일로 떨구기 또는 `claude`/`codex` 직접 호출). 코드 캡처 덕에 프롬프트는 이미 자기완결적 → 남은 건 "복사 안 하게" 하는 마지막 1인치. monacori를 *유일하게* 만드는 지점.
2. **변경에 집중** — 미변경 영역 접기(perf + 집중) + 공백 무시. `intellij-like-ui`를 "IntelliJ 패리티"가 아니라 *focus-on-changes*로 재스코핑.
3. **드리프트 정리** — 코드가 스펙을 앞질렀다. 스펙에 안 담긴 내비 capability(comment-box nav·Opt+화살표 단어이동·http 캐럿)는 "리뷰 내비게이션" 한 묶음으로 소급하거나 명시적으로 수용.
