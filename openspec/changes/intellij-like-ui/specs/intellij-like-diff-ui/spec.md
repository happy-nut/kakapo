## ADDED Requirements

### Requirement: 공백 무시
사용자는 공백을 무시하는 diff를 토글할 수 있어야 한다(SHALL). 켜져 있으면 공백만 다른 줄은 변경으로 표시되어서는 안 된다(MUST NOT).

#### Scenario: 공백 무시를 켜면 공백만 바뀐 변경이 사라진다
- **WHEN** 사용자가 "공백 무시"를 켤 때
- **THEN** 공백을 무시하고 diff를 다시 계산하며, 공백만 다른 줄은 더 이상 추가/삭제로 표시되지 않는다

#### Scenario: 끄면 전체 diff가 복원된다
- **WHEN** 사용자가 "공백 무시"를 끌 때
- **THEN** 공백만 바뀐 변경을 포함한 모든 차이가 다시 표시된다

### Requirement: 미변경 영역 접기
임계값보다 긴 미변경 구간은 숨긴 줄 수를 알려주는 펼침 가능한 형태로 접혀야 하며(SHALL), 요청 시 펼쳐질 수 있어야 한다(SHALL).

#### Scenario: 긴 미변경 구간이 접힌다
- **WHEN** 한 파일에 임계값보다 긴 미변경 줄 구간이 있을 때
- **THEN** 그 구간이 "… N unchanged lines …" 하나의 펼침 affordance로 접힌다

#### Scenario: 펼치면 숨긴 줄이 드러난다
- **WHEN** 사용자가 접힌 affordance를 펼칠 때
- **THEN** 숨겨졌던 미변경 줄들이 그 자리에 드러난다

#### Scenario: 변경된 줄과 그 컨텍스트는 항상 보인다
- **WHEN** 영역이 접혀 있을 때
- **THEN** 변경된 모든 줄과 그 주변 컨텍스트 줄은 항상 보인다(절대 숨겨지지 않는다)

### Requirement: 단어 단위 강조 토글
사용자는 줄 안(단어 단위) 변경 강조를 토글할 수 있어야 한다(SHALL).

#### Scenario: 단어 강조 토글
- **WHEN** 사용자가 단어 강조를 끌 때
- **THEN** 줄 안 단어 단위 강조가 사라지고 줄 단위 추가/삭제 색상만 남으며, 다시 켜면 단어 단위 강조가 복원된다

### Requirement: 카운터가 있는 차이 단위 이동
diff는 전체 차이 개수와 현재 위치를 보여주어야 하며(SHALL), 이동 시 현재 차이를 강조해야 한다(SHALL).

#### Scenario: 차이 카운터가 위치를 반영한다
- **WHEN** 사용자가 다음/이전 차이로 이동할 때(예: `F7` / `Shift+F7`)
- **THEN** 카운터가 "현재 인덱스 / 전체"로 갱신되고 현재 차이가 시각적으로 강조된다

#### Scenario: 양 끝에서 이동이 예측 가능하게 동작한다
- **WHEN** 사용자가 마지막 차이 뒤로(또는 첫 차이 앞으로) 넘어갈 때
- **THEN** 이동이 일관되게 동작한다(순환 또는 멈춤) — 오류 없이

### Requirement: 리뷰 옵션 설정 영속화
리뷰 옵션 선택값(공백 무시, 단어 강조, 미변경 접기)은 같은 repo 리뷰의 리로드 후에도 유지되어야 한다(SHALL).

#### Scenario: 선택값이 리로드 후에도 유지된다
- **WHEN** 사용자가 리뷰 옵션을 설정하고 같은 repo의 리뷰가 리로드될 때
- **THEN** 이전에 선택한 옵션들이 복원된다
