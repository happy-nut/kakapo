## ADDED Requirements

### Requirement: repo 크기와 무관한 즉시·무블록 오픈
리뷰는 repo 크기와 무관하게 즉시 열려야 하며(SHALL), 어떤 경우에도 메인 스레드를 길게 블록해서는 안 된다(MUST NOT). 리뷰가 표시되는 즉시 모든 키보드 단축키가 동작해야 한다(SHALL).

#### Scenario: 큰 repo를 열어도 블록되지 않는다
- **WHEN** 파일·diff 행이 매우 많은 repo의 리뷰를 열 때
- **THEN** UI가 즉시 표시되고, 거대한 DOM을 일괄 파싱·레이아웃하느라 멈추지 않는다

#### Scenario: 오픈 직후 단축키 동작
- **WHEN** 큰 repo 리뷰가 막 표시된 직후 사용자가 단축키를 누를 때
- **THEN** 해당 단축키가 정상 동작한다(메인 스레드가 점유되어 무시되지 않는다)

### Requirement: size-conditional 지연 모드
임계 이하의 리뷰는 현행 즉시 렌더 동작을 그대로 유지해야 한다(SHALL). 임계 초과 시에만 지연 렌더/로드 모드로 전환해야 한다(SHALL).

#### Scenario: 작은 repo는 현행 유지
- **WHEN** 파일·diff 행 수가 임계 이하인 리뷰를 열 때
- **THEN** 모든 파일 diff와 소스가 현행처럼 즉시 렌더되어 기존 동작이 변하지 않는다

#### Scenario: 큰 repo는 지연 모드
- **WHEN** 파일·diff 행 수가 임계를 초과하는 리뷰를 열 때
- **THEN** 파일은 접힌 헤더로 렌더되고 각 파일의 diff/소스는 필요 시 로드·머티리얼라이즈된다

### Requirement: 파일 diff·소스의 지연 머티리얼라이즈/로드
지연 모드에서 각 파일의 diff DOM은 그 파일이 펼쳐지거나 화면에 들어올 때 생성되어야 한다(SHALL). 파일 소스는 필요 시점(소스 뷰 열기·인덱스 빌드)에 on-demand로 로드되어야 하며(SHALL), 전체 소스를 초기 HTML에 일괄 임베드해서는 안 된다(MUST NOT).

#### Scenario: diff는 펼칠 때 생성
- **WHEN** 지연 모드에서 사용자가 접힌 파일을 펼치거나 그 파일이 스크롤로 화면에 들어올 때
- **THEN** 그 파일의 diff DOM이 그 시점에 생성되어 표시된다

#### Scenario: 소스는 필요할 때 로드
- **WHEN** 지연 모드에서 사용자가 한 파일의 소스 뷰를 열 때
- **THEN** 그 파일의 소스가 on-demand로 로드되어 표시된다(Electron=IPC, browser-serve=HTTP 엔드포인트)

### Requirement: 지연 모드에서의 기능 보존
지연 모드에서도 캐럿 네비게이션, 코멘트(질문/수정요청)·머지뷰, go-to-def(`Cmd/Ctrl+B`)가 정상 동작해야 한다(SHALL). 어떤 기능이 특정 파일을 대상으로 동작하기 전에 그 파일의 diff/소스가 준비되어야 한다(SHALL).

#### Scenario: go-to-def가 지연 소스를 로드
- **WHEN** 지연 모드에서 사용자가 아직 로드되지 않은 파일에 선언이 있는 심볼로 `Cmd/Ctrl+B` 점프할 때
- **THEN** 그 파일의 소스가 준비된 뒤 선언 위치로 이동한다(인덱스는 소스 보유 측에서 빌드)

#### Scenario: 미준비 파일로의 네비게이션
- **WHEN** 지연 모드에서 사용자가 아직 머티리얼라이즈되지 않은 파일로 이동·조작할 때
- **THEN** 그 파일이 먼저 머티리얼라이즈·로드된 뒤 동작이 수행된다
